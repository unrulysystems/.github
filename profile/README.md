# Unruly Systems

**Bold systems, made real.**

Unruly Systems builds native apps, games, and tools for ideas that do not fit the usual mold.

The studio is focused on practical experiments in native 3D, real-time interaction, React Native, Three.js/R3F, WebGPU/Dawn, physics, and E2E systems that run on real devices.

Unruly does not mean careless. It means unwilling to accept stale defaults when better systems are possible.

## Current Work

- Native WebGPU and Dawn experiments for React Native
- Renderer-neutral game architecture for Sortessori
- Native E2E infrastructure for React Native apps and games
- Physics integration paths that avoid fragile browser-only assumptions on native

## Repositories

- [`nativegpu`](https://github.com/unrulysystems/nativegpu): native rendering experiments for React Native, Dawn, WebGPU, Three.js, and R3F
- [`physics`](https://github.com/unrulysystems/physics): physics integration experiments for native React Native games and 3D apps
- [`r3f`](https://github.com/unrulysystems/r3f): React Three Fiber bridge experiments for native React Native renderers
- [`e2e`](https://github.com/unrulysystems/e2e): native E2E tooling experiments for React Native apps and games
- [`sortessori`](https://github.com/unrulysystems/sortessori): future home for Sortessori game experiments under Unruly Systems

Existing Sortessori and rn-playwright-driver development remains in the current working repositories until migration boundaries are explicit.
